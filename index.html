Requirements Analysis Document

Purpose

This document describes the current system in terms of functional and nonfunctional requirements, serving as a basis between the "client"(the conference analysis committee) and the developer.

Audience

The audience for the RAD includes the conference analysis committee, the subsequent readers and the developers who participate in the system design.

  1. Introduction

    While a lot of research does exist in the field of NLP sentiment analysis, this type of contextual sentiment analysis applied to conversation has reduced quantity and quality of related work.
    Despite this, we are confident that applying results and insight from the general field of sentiment analysis will prove to yield good results.

    1. Purpose of the system

      We consider using the Mixture of Experts model with a view to classifying the last reply of a three replies dialogue into one of the following categories: Happy, Sad, Angry, Others.
      We intend to use comittee machines, taking into account especially boosting algorithms, which proved to have the best results in the field recently.

    2. Scope of the system

      The system should be able to clasify the last reply of a three replies dialogue into one of the following categories: Happy, Sad, Angry or Others.

    3. Objectives and success criteria of the project
      • Outperforming known results on similar problems as well as the other teams participating in the SemEval-2019 workshop;
      • Respecting the deadline specified on the site of SemEval;
      • Decent training time and optimized time in terms of classification;
      • Scalability of the project;

  2. Proposed system
    1. Overview

      Our system should correctly classify replies as belonging to one of the following categories: 'happy', angry', 'sad', 'others'. Each of the entries from the training dataset should consist in a conversation, composed of three replies, with the flow as it follows: User1 -> User2 -> User1. There are 15k entries in the dataset labeled as 'happy', 'angry', 'sad' and 15k entries labeled as 'others'.
      For each entry, the system would apply a preprocessing algorithm, made up of more components, as described in the 'Implementation' section.
      Afterwards, the features which are set for that particular entries would be evaluated. In the following, the network would be fed with the information.
      From this point of view, there is applied the 'Mixture of Experts' technique, through which we combine neural networks and boosting algorithms, with particular weights for each of them.
      Multiple experiments will be conducted, with multiple algorithms, in order to see which of these combinations provides us with a model having the best accuracy. At the implementation level, we would have an abstract class entitled 'Model', so that each of the experiments involving different algorithms would implement the above class.

    2. Functional requirements ("shall lists")

      The system shall clasify the third reply from each entry of the dataset into one of the following categories:

      • 'happy'
      • 'angry'
      • 'sad'
      • 'others'

    3. Nonfunctional requirements
      1. Usability
        1. The classification system could be integrated in the applications meant to evaluate human mental health. For example, through one of these particular applications, the psychologist could have access to a set of conversations, where the replies would be labeled as 'happy', 'sad', 'angry' or 'others'. Consequently, by analysing these logs, it would be much easier for him to understand the causes of particular behaviours. Therapy sessions could therefore take place in a virtual environment.
        2. Secondly, this module could also be integrated into a social networking service, with a view to providing real-time emoticon suggestions for the conversations in the chats.
        3. Thirdly, when it comes to social networking applications, our classification system could be used in order to dynamically change the background of the chat window, according to the atmosphere implied by the conversation. This way, the application in question would respond to the people's general need for empathy.
        4. Moreover, having the replies in conversations labeled at the level of suggested emotions would facilitate providing suggestions for chat partners. For example, if there is logged a conversation with a high number of 'angry' and 'sad' labeled replies, the corresponding person would not be suggested anymore as a partner in chat. That would be a more appropriate system as compared to the virtual social networks, where there are highly suggested the persons with whom the users talks the most.
        5. The classification system could also be used to improve the efficiency of ads. For example, if according to the most recent conversations, the person is happy, there would be a bigger chance that she/he would buy a particular product. Also, if the replies show the person is sad or angry, there could be various products that could respond to the needs of that particular user.
        6. Another option would consist in using the results with a view to identifying false accounts. More exactly, if there is a new account, from which there are recorded many conversations labeled as angry and sad, this could represent a strong indicator that the account is false.
        7. Finally, for the dating apps, such as Tinder, there could be increased the level of safeness by using this classification system. For example, if most of the conversations of a user are offensive, there should be provided a negative score to that particular used, which would decrease his chances of being recommended to other users and eventually, if the context persists, the account might get to be blocked.
